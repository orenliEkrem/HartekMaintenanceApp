# Contributing to Hartek Maintenance App

Thank you for considering contributing!  
Please follow these guidelines to ensure smooth collaboration.

## 🧱 Development Setup

1. Fork the repository  
2. Clone your fork  
3. Create a new branch:
   ```bash
   git checkout -b feature/YourFeatureName
4. Make your changes
5. Test your changes
6. Commit with a clear message
7. Push and create a Pull Request

## 📌 Code Style

-Use meaningful class and method names

-Avoid duplicate code

-Keep UI logic separate from database logic

-Use try/catch blocks for all database operations

## 🔍 Pull Requests

-A good PR includes:

-Description of the changes

-Screenshots (if UI updated)

-Clear commit messages

Thank you for helping improve this project!
